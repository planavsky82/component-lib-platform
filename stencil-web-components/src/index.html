Get Started with Your Component Library Platform

This is the start point for your component library.

Before you start working ...

  1. Create your project and author name if you haven't done so already but editing your root-level package.json.
    2. {
     "name": "my-design-system",
     ...
     "author": "",
     ...
    }
  2. Also change the name property in the /stencil-web-components/package.json and /stencil-react-wrappers/package.json to something that fits your component library convention.
  3. Finally, make sure the componentCorePackage property in the /stencil-web-components/stencil.config.ts matches the name property in /stencil-web-components/package.json.

You can start learning how to develop the component library platform by editing this page (/stencil-web-components/src/index.html). The following assets will help:

  • Storybook for StencilJS Component Library
  • React Pattern Demo
  • TailwindCSS
  • Building a Micro Frontend

Building a Micro Frontend

The platform also can be used to generate a Micro Frontend, a more complex web component that is esentially its own micro-application and can be shared across other frontend applications. There is an example of a web component being used as a mirco frontend in the React Pattern Demo.
